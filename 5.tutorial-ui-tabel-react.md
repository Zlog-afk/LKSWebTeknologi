# Tutorial: Membuat UI Tabel di React

Tutorial ini membahas cara membuat komponen tabel yang reusable di React — mulai dari tabel statis paling dasar, sampai tabel dengan sorting, pagination, loading state, dan empty state. Contoh kasusnya memakai data "Produk" (cocok langsung dipakai untuk tampilan list di rancangan **Inventory Manager**), tapi polanya sama untuk Bookmark Manager maupun Expense Tracker.

---

## 1. Tabel Statis Paling Dasar

Mulai dari yang paling sederhana dulu: render array data jadi baris tabel.

```jsx
// components/ProductTable.jsx
function ProductTable({ products }) {
  return (
    <table>
      <thead>
        <tr>
          <th>SKU</th>
          <th>Nama</th>
          <th>Stok</th>
          <th>Harga</th>
        </tr>
      </thead>
      <tbody>
        {products.map((product) => (
          <tr key={product.id}>
            <td>{product.sku}</td>
            <td>{product.name}</td>
            <td>{product.stock}</td>
            <td>{product.price}</td>
          </tr>
        ))}
      </tbody>
    </table>
  );
}

export default ProductTable;
```

**Poin penting:**
- `key={product.id}` wajib ada di elemen `<tr>` paling luar hasil `.map()` — pakai `id` dari data, jangan pakai index array (index bikin bug kalau data di-sort/dihapus).
- Komponen ini "dumb" — cuma menerima data lewat props, tidak tahu dari mana data itu berasal. Ini bikin komponen gampang dipakai ulang di halaman lain.

## 2. Styling dengan Tailwind

```jsx
function ProductTable({ products }) {
  return (
    <table className="w-full border-collapse text-sm">

          <th className="p-3 border-b">SKU</th>

          <th className="p-3 border-b">Harga</th>
        </tr>
      </thead>
      <tbody>
        {products.map((product) => (
          <tr key={product.id} className="hover:bg-gray-50">
            <td className="p-3 border-b">{product.sku}</td>
            <td className="p-3 border-b">{product.name}</td>
            <td className="p-3 border-b">
              {product.stock <= product.minimum_stock ? (
                <span className="text-red-600 font-medium">⚠ {product.stock}</span>
              ) : (
                product.stock
              )}
            </td>

        ))}
      </tbody>
    </table>
  );
}
```

Badge "⚠" untuk stok menipis ini langsung mengikuti aturan `stock <= minimum_stock` yang sudah didefinisikan di skema database Inventory Manager sebelumnya.

## 3. Loading State & Empty State

Tabel jarang langsung punya data — biasanya nunggu fetch dari API dulu. Tangani 3 kondisi: loading, kosong, dan ada data.

```jsx
function ProductTable({ products, isLoading }) {
  if (isLoading) {
    return <p className="p-4 text-gray-500">Memuat data...</p>;
  }

  if (products.length === 0) {
    return <p className="p-4 text-gray-500">Belum ada produk.</p>;
  }

  return (
    <table className="w-full border-collapse text-sm">
      {/* ...sama seperti sebelumnya... */}
    </table>
  );
}
```

## 4. Menghubungkan ke API (fetch data asli)

```jsx
// pages/ProductListPage.jsx
import { useState, useEffect } from 'react';
import axios from '../lib/axios'; // instance axios dengan Bearer token
import ProductTable from '../components/ProductTable';

function ProductListPage() {
  const [products, setProducts] = useState([]);
  const [isLoading, setIsLoading] = useState(true);

  useEffect(() => {
    axios.get('/api/v1/products')
      .then((res) => setProducts(res.data.data))
      .finally(() => setIsLoading(false));
  }, []);

  return (
    <div className="p-6">
      <h1 className="text-xl font-semibold mb-4">Daftar Produk</h1>
      <ProductTable products={products} isLoading={isLoading} />
    </div>
  );
}

export default ProductListPage;
```

## 5. Sorting Kolom

Klik header kolom untuk sort ascending/descending.

```jsx
function ProductTable({ products }) {
  const [sortBy, setSortBy] = useState(null);
  const [sortDir, setSortDir] = useState('asc');

  const handleSort = (column) => {
    if (sortBy === column) {
      setSortDir(sortDir === 'asc' ? 'desc' : 'asc');
    } else {
      setSortBy(column);
      setSortDir('asc');
    }
  };

  const sortedProducts = [...products].sort((a, b) => {
    if (!sortBy) return 0;
    const valA = a[sortBy];
    const valB = b[sortBy];
    if (valA < valB) return sortDir === 'asc' ? -1 : 1;
    if (valA > valB) return sortDir === 'asc' ? 1 : -1;
    return 0;
  });

  const SortIcon = ({ column }) => {
    if (sortBy !== column) return null;
    return <span>{sortDir === 'asc' ? ' ▲' : ' ▼'}</span>;
  };

  return (
    <table className="w-full border-collapse text-sm">
      <thead>
        <tr className="bg-gray-100 text-left">
          <th className="p-3 border-b cursor-pointer" onClick={() => handleSort('name')}>
            Nama<SortIcon column="name" />
          </th>
          <th className="p-3 border-b cursor-pointer" onClick={() => handleSort('stock')}>
            Stok<SortIcon column="stock" />
          </th>
          <th className="p-3 border-b cursor-pointer" onClick={() => handleSort('price')}>
            Harga<SortIcon column="price" />
          </th>
        </tr>
      </thead>
      <tbody>
        {sortedProducts.map((product) => (
          <tr key={product.id}>
            <td className="p-3 border-b">{product.name}</td>
            <td className="p-3 border-b">{product.stock}</td>
            <td className="p-3 border-b">{product.price}</td>
          </tr>
        ))}
      </tbody>
    </table>
  );
}
```

**Catatan:** sorting di sini dilakukan di sisi frontend (client-side), cocok untuk data dengan jumlah baris tidak terlalu besar. Kalau datanya banyak (ribuan baris), sebaiknya sorting dilakukan di backend lewat parameter query, misalnya `GET /api/v1/products?sort_by=stock&sort_dir=asc`.

## 6. Pagination

### 6.1 Pagination Sisi Frontend (untuk data kecil)

```jsx
function ProductTable({ products }) {
  const [page, setPage] = useState(1);
  const perPage = 5;

  const totalPages = Math.ceil(products.length / perPage);
  const paginatedProducts = products.slice((page - 1) * perPage, page * perPage);

  return (
    <div>
      <table className="w-full border-collapse text-sm">
        <tbody>
          {paginatedProducts.map((product) => (
            <tr key={product.id}>
              <td className="p-3 border-b">{product.name}</td>
              <td className="p-3 border-b">{product.stock}</td>
            </tr>
          ))}
        </tbody>
      </table>

      <div className="flex justify-between items-center mt-3">
        <button
          disabled={page === 1}
          onClick={() => setPage(page - 1)}
          className="px-3 py-1 border rounded disabled:opacity-40"
        >
          Sebelumnya
        </button>
        <span>Halaman {page} dari {totalPages}</span>
        <button
          disabled={page === totalPages}
          onClick={() => setPage(page + 1)}
          className="px-3 py-1 border rounded disabled:opacity-40"
        >
          Berikutnya
        </button>
      </div>
    </div>
  );
}
```

### 6.2 Pagination Sisi Backend (lebih tepat untuk data besar)

Karena endpoint di rancangan sebelumnya (`GET /api/v1/products`) pakai `paginate()` bawaan Laravel, response-nya sudah mengandung `meta` (current_page, last_page, total). Frontend tinggal minta halaman sesuai state:

```jsx
useEffect(() => {
  axios.get(`/api/v1/products?page=${page}`)
    .then((res) => {
      setProducts(res.data.data);
      setTotalPages(res.data.meta.last_page);
    });
}, [page]);
```

Pola ini lebih tepat dipakai untuk Inventory Manager dan Expense Tracker yang datanya bisa terus bertambah, dibanding sorting/pagination manual di frontend.

## 7. Komponen Tabel Reusable (Generik)

Kalau tabel serupa dipakai berkali-kali (Bookmark, Produk, Transaksi), buat komponen generik yang menerima definisi kolom lewat props:

```jsx
// components/DataTable.jsx
function DataTable({ columns, data, isLoading, emptyMessage = 'Tidak ada data.' }) {
  if (isLoading) return <p className="p-4 text-gray-500">Memuat data...</p>;
  if (data.length === 0) return <p className="p-4 text-gray-500">{emptyMessage}</p>;

  return (
    <table className="w-full border-collapse text-sm">
      <thead>
        <tr className="bg-gray-100 text-left">
          {columns.map((col) => (
            <th key={col.key} className="p-3 border-b">{col.label}</th>
          ))}
        </tr>
      </thead>
      <tbody>
        {data.map((row) => (
          <tr key={row.id} className="hover:bg-gray-50">
            {columns.map((col) => (
              <td key={col.key} className="p-3 border-b">
                {col.render ? col.render(row) : row[col.key]}
              </td>
            ))}
          </tr>
        ))}
      </tbody>
    </table>
  );
}

export default DataTable;
```

**Cara pakai untuk tabel Transaksi (Expense Tracker):**

```jsx
const columns = [
  { key: 'transaction_date', label: 'Tanggal' },
  { key: 'category', label: 'Kategori', render: (row) => row.category.name },
  {
    key: 'amount',
    label: 'Jumlah',
    render: (row) => (
      <span className={row.type === 'income' ? 'text-green-600' : 'text-red-600'}>
        {row.type === 'income' ? '+' : '−'} Rp{row.amount.toLocaleString('id-ID')}
      </span>
    ),
  },
];

<DataTable columns={columns} data={transactions} isLoading={isLoading} />
```

**Cara pakai untuk tabel Bookmark:**

```jsx
const columns = [
  { key: 'title', label: 'Judul' },
  { key: 'url', label: 'URL' },
  { key: 'category', label: 'Kategori' },
  {
    key: 'is_favorite',
    label: 'Favorit',
    render: (row) => (row.is_favorite ? '★' : '☆'),
  },
];

<DataTable columns={columns} data={bookmarks} isLoading={isLoading} />
```

Satu komponen `DataTable` ini bisa dipakai ulang untuk ketiga aplikasi (Bookmark Manager, Inventory Manager, Expense Tracker) — tinggal ganti definisi `columns`-nya saja.

## 8. Ringkasan Urutan Belajar

1. Tabel statis dari array (`.map()` + `key`)
2. Styling dasar dengan Tailwind
3. Loading state & empty state
4. Fetch data asli dari API (`useEffect` + axios)
5. Sorting kolom (klik header)
6. Pagination (frontend untuk data kecil, backend untuk data besar)
7. Generalisasi jadi komponen `DataTable` reusable lewat props `columns`

Kalau latihan untuk kompetisi, cukup kuasai langkah 1–4 dulu sampai lancar (itu yang paling sering dipakai di soal CRUD), baru lanjut ke sorting/pagination/reusable component sebagai nilai tambah.
